https://developer.mozilla.org/en-US/docs/Web/HTTP/Status
Client error responsesSection
200 server is ok
400 Bad Request: This response means that server could not understand the request due to invalid syntax.    		 
401 Unauthorized: Although the HTTP standard specifies "unauthorized", semantically this response means "unauthenticated".
    That is, the client must authenticate itself to get the requested response.
403 – Server is trying to access the restricted file.	
404 Not Found: The server can not find requested resource. In the browser, this means the URL is not recognized.
    In an API, this can also mean that the endpoint is valid but the resource itself does not exist	 


Server error responsesSection
500 Internal Server Error :The server has encountered a situation it doesn't know how to handle.
502 Bad Gateway :This error response means that the server, while working as a gateway to get a response needed to handle the request,
    got an invalid response.
503 –Server is busy.
